**Programs** are the primary software primitive on Modaic Hub.

![Programs on Modaic Hub](https://d1pz4mbco29rws.cloudfront.net/public/repository.png)

A program repository stores versioned component(s) of an AI system rather than just a raw checkpoint. In practice, that means the code, configuration, and serialized artifacts needed to run a declarative AI system.

## What belongs in a program repository

A program repository includes a few required files:

- `pyproject.toml`
- `program.json`
- `config.json`

Most program repositories should also include:

- Program source code
- A root `README.md`
- Supporting assets and documentation
- Version history across branches, tags, and commits

`program.json` is the serialized DSPy program artifact, before or after optimization. `config.json` is the serialized state for the program's environment, experiment-specific parameters, and DSPy signature.

## Addressing a program

Programs are referenced by repository ID:

```text
entity-name/program-name
```

This is the identifier you use when loading a program from Modaic Hub.

## Loading programs from the Hub

Use `AutoProgram` to load a precompiled program from Hub:

```python
from modaic import AutoProgram

program = AutoProgram.from_precompiled("entity-name/program-name")
```

You can also target a specific revision:

```python
program = AutoProgram.from_precompiled(
    "entity-name/program-name",
    rev="v1.0"
)
```

If the repository contains a precompiled `modaic.Predict`, load it directly with `Predict.from_precompiled(...)`:

```python
from modaic import Predict

predict = Predict.from_precompiled("entity-name/program-name")
```

See [Auto Classes](/guides/basic_usage/auto_program) for the SDK-side loading workflow.

For packaging and workspace layout details, see [Bundling a Program](/guides/basic_usage/bundling_a_program).
