Program repositories on Modaic Hub are versioned Git repositories.

The **Files and Versions** surface is where you browse repository contents, inspect history, and reference a specific revision of a program.

![Files and Versions](https://d1pz4mbco29rws.cloudfront.net/public/repo-files.png)

## What you can do

From the repository file views, you can:

- Browse the file tree
- Open and inspect individual files
- View commit history
- Reference a repository at a specific branch, tag, or commit

For programs, that file history includes both source code and the required serialized artifacts:

- `pyproject.toml`
- `program.json`
- `config.json`

## Revisions

Programs can be loaded from a specific revision by passing `rev`:

```python
from modaic import AutoProgram

program = AutoProgram.from_precompiled(
    "entity-name/program-name",
    rev="main"
)
```

The `rev` value can be:

- A branch name
- A tag
- A commit hash

For example:

```python
program = AutoProgram.from_precompiled("entity-name/program-name", rev="v1.0")
program = AutoProgram.from_precompiled("entity-name/program-name", rev="918ad95")
```

## Editing and updates

You can update repository contents through the web file editor or by pushing changes through Git-based workflows.

In either case, the repository history remains part of the program itself, which makes changes auditable and reversible at the revision level.

This matters for programs because code and serialized artifacts evolve together. A revision does not just represent a code snapshot. It can also represent:

- A newly optimized DSPy program in `program.json`
- A changed environment or signature state in `config.json`
- Updated dependencies in `pyproject.toml`

## What belongs in this history

Treat files and versions as the source of truth for:

- Program code changes
- README and program card updates
- Serialized program artifact updates
- Config and signature-state updates
- Dependency changes
- Released artifacts tied to a specific revision
