## What is an Arbiter

An arbiter is an aligned LLM classifier, extractor, or judge that is acutely aware of it's own confidence. Modaic uses the latest in **Mechanistic Interpretability** to get highly calibrated confidence scores for each prediction an arbiter makes. Some tasks you can use an arbiter for are.

- **Subjective Rating**: *"Rate this AI response on a scale of 1-5"*
- **Binary Classification**: *"Determine the relevance of this article for my research on polar bears"*
- **Multi-Class Classification**: *"Determine the content risk level: safe, mild, moderate, or severe"*
- **Multi-Label Classification**: *"Extract the user intent, tone, and topic from this support message"*
- **Triage + Routing**: *"Determine if this user message requires a human response, an AI response, or no response"*

### Using the Modaic SDK
Install the Modaic SDK and create your arbiter locally.

First, install the Modaic SDK with [uv](https://docs.astral.sh/uv/):

```bash
uv add modaic
```

Then, create your arbiter:

<Info>
  Make sure you have a Modaic token set in your environment variables. You can
  get one [here](https://modaic.dev/settings/tokens).
  ![Create Token](https://d1pz4mbco29rws.cloudfront.net/public/create_token.png)
</Info>

<Info>
  For whichever provider you use, make sure you securely set the environment
  variables needed to run that model. You can manage them
  [here](https://www.modaic.dev/settings/env-vars). For the Together model in
  this example, set `TOGETHER_API_KEY` so Modaic can run prompt optimizations.![Environment Variables](https://d1pz4mbco29rws.cloudfront.net/public/env_vars.png)
</Info>



```python
import dspy
import modaic


class CodeCompletionSignature(dspy.Signature):
    """Given a prompt and a code completion in luau, Roblox's scripting language,
    evaluate the quality and relevance of the code completion in relation to the prompt.
    """

    prompt: str = dspy.InputField(desc="The prompt used to generate the code completion")
    completion: str = dspy.InputField(desc="The code completion to evaluate")
    quality: int = dspy.OutputField(desc="The quality of the completion (1-4)")


if __name__ == "__main__":
    code_completion_arbiter = modaic.Predict(
        CodeCompletionSignature,
        lm=dspy.LM(model="together_ai/Qwen/Qwen3-VL-32B-Instruct"),
    ).as_arbiter()

    code_completion_arbiter.push_to_hub("roblox/code-completions", private=True)
```

## Supported Models

Modaic currently has publicly available probes for these models:

- `Qwen/Qwen3-32B`
- `Qwen/Qwen3-VL-32B-Instruct`
- `Qwen/Qwen3.5-4B`
- `openai/gpt-oss-120b`

If you'd like to use your own model or a private beta model, contact us
[here](https://www.modaic.dev/contact).

## Running your Arbiter

Once your arbiter is pushed to modaic hub you can run it via Modaic's API.

<CodeGroup>
```python python
from modaic import Arbiter

# Load the arbiter from the same repo you pushed it to
code_completion_arbiter = Arbiter("roblox/code-completions")

# Call your arbiter, kwargs should match the fields you declared in the dspy.Signature
result = code_completion_arbiter(prompt="Print hello world", completion="print('Hello World')")

# Access the output
quality = result.output.quality

print("Quality:", quality)
```

```javascript javascript
const response = await fetch("https://api.modaic.dev/api/v1/arbiters/predictions", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${process.env.MODAIC_TOKEN}`,
  },
  body: JSON.stringify({
    input: {
      prompt: "Print hello world",
      completion: "print('Hello World')",
    },
    arbiters: [
      {
        arbiter_repo: "roblox/code-completions",
      },
    ],
  }),
});

const data = await response.json();
const quality = data.predictions[0].output.quality;

console.log("Quality:", quality);
```

curl -X POST https://api.modaic.dev/api/v1/arbiters/predictions \

    "input": {

      "completion": "print(\"Hello World\")"
    },

      {
        "arbiter_repo": "roblox/code-completions",
      }
    ]
  }'
