The Modaic Python SDK is how you build arbiters, run them, and manage their examples from Python. It ships in two packages:

- **`modaic`** — the full SDK. Use it to *create and push* arbiters with `dspy.Signature` and `modaic.Predict`, plus everything the client does.
- **`modaic-client`** — a lightweight, dependency-light client for production services, serverless functions, and restrictive sandboxes that only need to *call* arbiters and manage examples. It ships without DSPy or litellm, so cold starts are fast.

`ModaicClient` is the thin HTTP client shared by both packages — use it to run arbiters, ingest and annotate examples, and poll jobs.

## Installation

There are two ways to install the client depending on what else you need.

<Tabs>
<Tab title="Full SDK">
Pick this when you are building, training, or bundling programs with DSPy — it includes `ModaicClient` plus `modaic.Predict`, `PrecompiledProgram`, `AutoProgram`, and the batch stack.

```bash
pip install modaic
```

```python
from modaic import ModaicClient

client = ModaicClient(modaic_token="...")
```
</Tab>
<Tab title="Client Only">
Pick this for production services, serverless functions, or restrictive sandboxes where you only need to *call* arbiters. It ships without DSPy or litellm, so cold starts are fast and the dependency surface is minimal.

```bash
pip install modaic-client
```

```python
from modaic_client import ModaicClient

client = ModaicClient(modaic_token="...")
```
</Tab>
</Tabs>

The public API is identical across both packages — only the import path and dependency footprint differ.

## Authentication

The token can be passed explicitly or picked up from the `MODAIC_TOKEN` environment variable:

```python
import os
os.environ["MODAIC_TOKEN"] = "..."

client = ModaicClient()  # resolves token from env
```

## Creating an Arbiter

<Note>
Creating an arbiter requires the **full SDK** (`pip install modaic`), not the
lightweight `modaic-client` package. `modaic-client` can only *call* arbiters and
manage examples — building one needs `dspy.Signature` and `modaic.Predict`, which
ship with `modaic`.
</Note>

Define the arbiter's interface with a `dspy.Signature`, wrap it in `modaic.Predict`, call `.as_arbiter()`, then push it to Modaic Hub. Output fields must use `Literal` (or another enumerable type) so the arbiter has a finite space to calibrate against.

```python create_arbiter.py
from typing import Literal

import dspy
import modaic


class SupportTriageSignature(dspy.Signature):
    """Decide how a support ticket should be handled."""

    ticket: str = dspy.InputField(desc="The incoming support ticket")
    action: Literal["refund", "faq", "escalate"] = dspy.OutputField(
        desc="How to handle the ticket"
    )


if __name__ == "__main__":
    arbiter = modaic.Predict(
        SupportTriageSignature,
        lm=dspy.LM(model="together_ai/openai/gpt-oss-120b"),
    ).as_arbiter()

    arbiter.push_to_hub("your-org/support-triage", private=True)
```

<Info>
  Set the provider API key for your arbiter's model as an Environment Variable on
  [Modaic Hub](https://www.modaic.dev/settings/env-vars). The example above uses
  Together, so set `TOGETHER_API_KEY`. **Always call `.as_arbiter()` before
  `push_to_hub`** — otherwise the repo won't be recognized as an arbiter.
</Info>

Once pushed, call it with `Arbiter` as shown below.

## Calling an Arbiter

The preferred pattern is to construct an `Arbiter` directly — it discovers the shared `ModaicClient` automatically, so there's no need to call `client.get_arbiter(...)`.

### Single Call

Use `Arbiter.predict` to run an arbiter on a single input. Inputs are passed as keyword arguments matching the arbiter's signature.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

prediction = arbiter.predict(ticket="My payment failed twice in a row.")

print(prediction.output)
print(prediction.reasoning)
```

Example output:

```text
{'action': 'refund'}
The user reports a duplicate failed charge, which the rubric classifies as a billing issue eligible for refund.
```

### Multiple Examples

Use `Arbiter.predict_all` to run one arbiter against many examples in a single batch job. The call dispatches a server-side batch and, by default, blocks until results are ready.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

results = arbiter.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice in a row."}},
        {"input": {"ticket": "How do I change my plan?"}},
        {"input": {"ticket": "The app crashes when I open settings."}},
    ],
)

for row in results:
    pred = row.predictions[0]
    print(row.example_id, pred.output)
```

Example output:

```text
ex_01HZ9K2F8V {'action': 'refund'}
ex_01HZ9K2F8W {'action': 'faq'}
ex_01HZ9K2F8X {'action': 'escalate'}
```

Pass `wait_for=None` to get back a `BatchJob` handle you can poll with `job.status()` or `job.wait(...)`.

### Re-predicting on existing examples

Pass `example_ids` instead of `examples` to re-run predictions on examples you've already ingested. The server fails fast (400) if any id doesn't exist for a requested arbiter; the new predictions land as a fresh version on the existing example rows.

```python
results = arbiter.predict_all(
    example_ids=["ex_01HZ9K2F8V", "ex_01HZ9K2F8W"],
)
```

### Batch Confidence Scoring

Pass `compute_confidence=True` to `predict_all` to kick off batch confidence scoring after every prediction in the batch is persisted. Scoring is filtered to *this job's* predictions — it won't touch unrelated NULL-confidence rows in the same repo.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

results = arbiter.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice."}},
        {"input": {"ticket": "Thanks, your team is amazing!"}},
    ],
    compute_confidence=True,
    wait_for="scores",   # block until scoring finishes; defaults to "predictions"
)

for row in results:
    for pred in row.predictions:
        print(pred.output, pred.confidence)
```

`wait_for` controls how long `predict_all` blocks:

- `"predictions"` (default) — return as soon as predictions are persisted. Predictions are usable immediately; reading `pred.confidence` then blocks on the still-running score.
- `"scores"` — block until scoring finishes. Requires `compute_confidence=True`. Returned predictions have `confidence` already populated.
- `None` — return a `BatchJob` handle right away without waiting at all.

<Warning>
Batch confidence scoring is billed per prediction and is cheaper than online (single-prediction) confidence scoring. Use it whenever you don't need a confidence score on the response path.
</Warning>

### Multiple Arbiters

Use `ModaicClient.predict_all` when you need to **run multiple arbiters against the same set of examples**. All arbiters are dispatched in a single batch job and evaluated concurrently on the server.

```python
from modaic_client import ModaicClient, Arbiter

client = ModaicClient()

triage = Arbiter("your-org/support-triage")
sentiment = Arbiter("your-org/sentiment")

results = client.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice in a row."}},
        {"input": {"ticket": "Thanks, your team is amazing!"}},
    ],
    arbiters=[triage, sentiment],
)

for row in results:
    for pred in row.predictions:
        print(row.example_id, pred.arbiter_repo, pred.output)
```

Example output:

```text
ex_01HZ9K2F8V your-org/support-triage {'action': 'refund'}
ex_01HZ9K2F8V your-org/sentiment {'label': 'negative'}
ex_01HZ9K2F8W your-org/support-triage {'action': 'faq'}
ex_01HZ9K2F8W your-org/sentiment {'label': 'positive'}
```

### Online Confidence Scoring

Pass `compute_confidence=True` to `Arbiter.predict` to kick off confidence scoring as soon as the prediction completes. Reading `prediction.confidence` then blocks only on whatever work is left, instead of starting from scratch.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

prediction = arbiter.predict(
    ticket="My payment failed twice in a row.",
    compute_confidence=True,
)

print(prediction.output)
print(prediction.confidence)
```

Example output:

```text
{'action': 'refund'}
0.87
```

<Warning>
Online confidence scoring is billed per call. Only enable `compute_confidence=True` when you need a confidence score in the response path — for offline scoring of stored predictions, use the batch confidence APIs instead.
</Warning>

## Ingest examples

```python
response = arbiter.ingest_examples([
    {"input": {"ticket": "My payment failed twice in a row."}, "ground_truth": "refund"},
    {"input": {"ticket": "How do I change my plan?"}, "ground_truth": "faq"},
])

print(response)
```

Example output:

```text
queued=True example_ids=['ex_01HZ9K2F8V', 'ex_01HZ9K2F8W']
```

## List examples

```python
page = arbiter.list_examples(page=1, page_size=50)

print(page.total, page.page, page.total_pages)
for example in page.items:
    print(example.id, example.ground_truth, example.confidence)
```

Example output:

```text
128 1 3
ex_01HZ9K2F8V refund 0.87
ex_01HZ9K2F8W faq 0.92
...
```

## Annotate examples

```python
response = arbiter.annotate_example(
    "ex_01HZ9K2F8V",
    ground_truth="escalate",
    ground_reasoning="Repeated failures suggest a payment-provider outage, not a refund case.",
)

print(response)
```

Example output:

```text
status='ok'
```
