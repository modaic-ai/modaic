The Modaic TypeScript SDK lets you create arbiters, run them, and update them from TypeScript or JavaScript. The `Arbiter` class is a thin wrapper over the Modaic REST API and git — it never runs an LLM locally. `predict()` calls the API and the Modaic server runs the model; `create()` and `update()` write the arbiter's signature schema and stored prompt and push them to Modaic Hub. These files are produced the same way as the [Python SDK](/docs/sdks/python_sdk), so judges authored in either language interoperate.

## Installation

<CodeGroup>
```bash npm
npm add modaic zod
```

```bash pnpm
pnpm add modaic zod
```

```bash bun
bun add modaic zod
```

```bash yarn
yarn add modaic zod
```
</CodeGroup>

Signatures are defined with [Zod](https://zod.dev), so install it alongside the SDK.

<Note>
This package is published as an ES Module (ESM) only and targets modern
JavaScript runtimes (Node.js 18+, Bun, Deno). For CommonJS projects, load it with
a dynamic `await import("modaic")`.
</Note>

## Authentication

The SDK reads your access token from the `MODAIC_TOKEN` environment variable. Generate one [here](https://modaic.dev/settings/tokens).

```bash
export MODAIC_TOKEN=...
```

You can also pass `access_token` explicitly to `Arbiter.create` / `arbiter.update`. The SDK is configured through these environment variables:

| Env var          | Default                   | Purpose                                  |
| ---------------- | ------------------------- | ---------------------------------------- |
| `MODAIC_TOKEN`   | —                         | Access token (or pass `access_token`)    |
| `MODAIC_API_URL` | `https://api.modaic.dev`  | Modaic REST API base URL                 |
| `MODAIC_GIT_URL` | `https://git.modaic.dev`  | Modaic Hub git host                      |
| `MODAIC_CACHE`   | `~/.cache/modaic`         | Staging directory for git working trees  |

## Creating an Arbiter

Define what the arbiter sees (`input`) and what it decides (`output`) with a `Signature`, backed by Zod schemas. Then call `Arbiter.create` to write the arbiter's `config.json` (signature schema) and `program.json` (stored prompt) and push them to Modaic Hub. Arbiters are private by default.

```typescript create-arbiter.ts
import { Arbiter, Signature } from "modaic";
import { z } from "zod";

const signature = new Signature({
  instructions: "Decide whether the answer correctly addresses the question.",
  input: z.object({
    question: z.string().describe("The user's question"),
    answer: z.string().describe("The answer to judge"),
  }),
  output: z.object({
    verdict: z.string().describe("correct | incorrect"),
  }),
});

const arbiter = await Arbiter.create({
  repo: "your-org/quality-judge",
  signature,
  // The LiteLLM model string the Modaic server runs this arbiter with,
  // in "<provider>/<model>" form.
  model: "together_ai/openai/gpt-oss-120b",
  commit_message: "initial judge",
  private: true,
});

console.log(`Pushed to ${arbiter.repo} (rev: ${arbiter.rev})`);
```

<Info>
  Set the provider API key for your arbiter's model as an Environment Variable on
  [Modaic Hub](https://www.modaic.dev/settings/env-vars). The example above uses
  Together, so set `TOGETHER_API_KEY`. `model` is required — an arbiter with no
  model can't be run.
</Info>

## Running an Arbiter

Construct a handle to an existing arbiter with `new Arbiter("owner/name")` and call `predict()`. No network call happens on construction — `predict()` runs the arbiter server-side over a single HTTP request. Inputs must match the fields declared on the signature.

```typescript predict.ts
import { Arbiter } from "modaic";

const arbiter = new Arbiter("your-org/quality-judge");

const result = await arbiter.predict({
  question: "What is the capital of France?",
  answer: "Paris is the capital of France.",
});

console.log("verdict:  ", result.output?.verdict);
console.log("reasoning:", result.reasoning);
```

Open a specific revision by passing `rev` (a branch, tag, or commit):

```typescript
const arbiter = new Arbiter("your-org/quality-judge", { rev: "v1" });
```

`predict()` accepts a second options argument for `ground_truth`, `ground_reasoning`, and `compute_confidence`. Pass `compute_confidence: true` to request a calibrated confidence score for the prediction (scored asynchronously after the prediction is persisted).

## Updating an Arbiter

Push a new revision with `update()`. Pass a new `signature` (and `model`) to rewrite the schema and prompt, or just `metadata` to update the arbiter card.

```typescript
await arbiter.update({
  signature,
  model: "together_ai/openai/gpt-oss-120b",
  commit_message: "tweak prompt",
});
```

## Low-level REST client

For direct, typed access to every Modaic REST endpoint (chat completions, jobs, examples, batch predictions), use `ModaicClient`.

```typescript
import { ModaicClient } from "modaic";

const client = new ModaicClient({
  token: process.env.MODAIC_TOKEN ?? "",
});

const result = await client.createPredictionApiV2ArbitersPredictionsPost({
  input: { question: "...", answer: "..." },
  arbiterRepo: "your-org/quality-judge",
});
```
