Running an arbiter over many examples is a **batch job**. `predict_all` dispatches
the job server-side and, by default, blocks until results are ready; pass
`wait_for=None` to get back a `BatchJob` handle you can poll, stream, or cancel.

## Running a batch

`Arbiter.predict_all` runs one arbiter against many examples in a single job.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

results = arbiter.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice in a row."}},
        {"input": {"ticket": "How do I change my plan?"}},
        {"input": {"ticket": "The app crashes when I open settings."}},
    ],
)

for row in results:
    pred = row.predictions[0]
    print(row.example_id, pred.output)
```

Example output:

```text
ex_01HZ9K2F8V {'action': 'refund'}
ex_01HZ9K2F8W {'action': 'faq'}
ex_01HZ9K2F8X {'action': 'escalate'}
```

### `predict_all` parameters

| Parameter            | Type                                | Default           | Description                                                        |
| -------------------- | ----------------------------------- | ----------------- | ----------------------------------------------------------------- |
| `examples`           | `list[BatchExample]`                | `None`\*          | New examples to ingest and predict.                               |
| `example_ids`        | `list[str]`                         | `None`\*          | Existing example ids to re-predict.                               |
| `compute_confidence` | `bool`                              | `False`           | Score predictions after they persist.                             |
| `wait_for`           | `"predictions" \| "scores" \| None` | `"predictions"`   | How long to block (see below).                                    |
| `poll_interval`      | `float`                             | `30.0`            | Seconds between status polls when blocking.                       |
| `timeout`            | `float`                             | `3600.0`          | Max seconds to block.                                             |
| `show_progress`      | `bool`                              | `True`            | Render a progress bar while blocking.                             |
| `on_event`           | `Callable[[BatchProgressEvent], None]` | `None`         | Callback invoked on every progress event.                        |

\* Provide exactly one of `examples` or `example_ids`.

Each `BatchExample` is a dict:

| Key                | Type                  | Required | Description                       |
| ------------------ | --------------------- | -------- | --------------------------------- |
| `input`            | `dict`                | yes      | The input fields for the arbiter. |
| `alt_id`           | `str`                 | no       | Your own external id.             |
| `ground_truth`     | `dict`                | no       | Known-correct output.             |
| `ground_reasoning` | `str`                 | no       | Explanation for the ground truth. |
| `split`            | `"train" \| "test"`   | no       | Dataset split tag.                |

Pass `example_ids` instead of `examples` to re-run predictions on examples you've
already ingested. The server fails fast (400) if any id doesn't exist for a requested
arbiter; the new predictions land as a fresh version on the existing example rows.

```python
results = arbiter.predict_all(example_ids=["ex_01HZ9K2F8V", "ex_01HZ9K2F8W"])
```

## Controlling when `predict_all` returns

`wait_for` controls how long the call blocks:

- `"predictions"` (default) — return as soon as predictions are persisted.
  Predictions are usable immediately; reading `pred.confidence` then blocks on the
  still-running score.
- `"scores"` — block until scoring finishes. Requires `compute_confidence=True`.
  Returned predictions have `confidence` already populated.
- `None` — return a `BatchJob` handle right away without waiting at all.

## Batch confidence scoring

Pass `compute_confidence=True` to score every prediction in the batch after it
persists. Scoring is filtered to *this job's* predictions.

```python
results = arbiter.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice."}},
        {"input": {"ticket": "Thanks, your team is amazing!"}},
    ],
    compute_confidence=True,
    wait_for="scores",   # block until scoring finishes
)

for row in results:
    for pred in row.predictions:
        print(pred.output, pred.confidence)
```

<Warning>
Batch confidence scoring is billed per prediction and is cheaper than online
(single-prediction) confidence scoring. Use it whenever you don't need a confidence
score on the response path.
</Warning>

## The `BatchJob` handle

Pass `wait_for=None` to get a `BatchJob` you can drive yourself:

```python
job = arbiter.predict_all(examples=[...], wait_for=None)

print(job.status())                 # current snapshot (dict)
for event in job.events():          # stream progress over SSE
    print(event.event, event.status)
results = job.results()             # fetch all results once finished
```

| Method                                                                   | Returns                     | Description                                                            |
| ------------------------------------------------------------------------ | --------------------------- | -------------------------------------------------------------------- |
| `status()`                                                               | `dict`                      | Current job status snapshot.                                         |
| `events(*, timeout=3600.0)`                                              | `Iterator[BatchProgressEvent]` | Stream progress snapshots; ends on the `finish` event.            |
| `results()`                                                              | `list[BatchExampleResult]`  | All per-example results from the completed job.                     |
| `wait(poll_interval=30.0, timeout=3600.0, *, wait_for="predictions", show_progress=True, on_event=None)` | `list[BatchExampleResult]` | Block until the requested milestone, then return results. |
| `cancel()`                                                               | `dict`                      | Cancel an in-flight job.                                            |

A `BatchProgressEvent` carries `event` (`start` / `prediction` / `score` /
`finish`), `status` (`predicting` / `scoring` / `done` / `failed`), per-phase
progress counters, and (on the terminal event) a results summary or an `error`.

Each `BatchExampleResult` has `example_id`, `input`, and `predictions` (one
`ArbiterPrediction` per arbiter in the batch).
