## 1. Install Modaic

Install the Modaic SDK into your project. Use `uv` or `pip` for the [Python SDK](/docs/python_sdk/installation), or `npm` for the [TypeScript SDK](/docs/typescript_sdk/installation).

<CodeGroup>
```bash uv
uv add modaic
```

```bash pip
pip install modaic
```

```bash npm
npm add modaic zod
```

</CodeGroup>

Set your Modaic token so the SDK can authenticate with Modaic Hub. Generate one [here](https://modaic.dev/settings/tokens).

```bash
export MODAIC_TOKEN=...
```

<Info>
  Modaic manages inference for supported models out of the box — no provider key
  needed. To bring your own inference provider for the same model, set that
  provider's API key on [Modaic
  Platform](https://www.modaic.dev/settings/env-vars) and reference the same
  model id with the provider's prefix. The example below uses
  `together_ai/openai/gpt-oss-120b`, so set `TOGETHER_API_KEY`.
</Info>

## 2. Install the Modaic Skills

Modaic ships a set of skills for coding agents (Claude Code, Cursor, Codex, etc.). The skills teach your agent how to define signatures, create Arbiters, push them to Modaic Hub, and debug common failures.

Run this from the root of your project.

```bash
npx skills add modaic-ai/modaic
```

This installs the `modaic-python-sdk`, `modaic-typescript-sdk`, `modaic-hub`, `modaic-api`, and `modaic-troubleshooting` skills into your agent's skill directory.

## 3. Create your first Arbiter

An **Arbiter** is an LLM judge that returns, a decision, it's reasoning and a calibrated confidence score. In Python, define one with a `dspy.Signature` and `modaic.Predict`; in TypeScript, define a `Signature` with [Zod](https://zod.dev) and call `Arbiter.create`.

<CodeGroup>
```python create_arbiter.py
import dspy
import modaic

class CodeCompletionSignature(dspy.Signature):
    """Given a prompt and a code completion in Luau, Roblox's scripting language,
    evaluate the quality and relevance of the code completion in relation to the prompt.

    Use a 1-4 scale:
      1 - irrelevant or broken
      2 - relevant but incorrect
      3 - mostly correct, minor issues
      4 - correct and idiomatic
    """

    prompt: str = dspy.InputField(desc="The prompt used to generate the completion")
    completion: str = dspy.InputField(desc="The code completion to evaluate")
    quality: modaic.Scale[1, 4] = dspy.OutputField(
        desc="Quality score from 1 (worst) to 4 (best)"
    )


if __name__ == "__main__":
    arbiter = modaic.Predict(
        CodeCompletionSignature,
        lm=dspy.LM(model="together_ai/openai/gpt-oss-120b"),
    ).as_arbiter()

    arbiter.push_to_hub("your-org/code-completions")
```

```typescript create_arbiter.ts
import { Arbiter, Signature, Scale } from "modaic";
import { z } from "zod";

const signature = new Signature({
  instructions: `Given a prompt and a code completion in Luau, Roblox's scripting language, evaluate the quality and relevance of the code completion in relation to the prompt.

Use a 1-4 scale:
  1 - irrelevant or broken
  2 - relevant but incorrect
  3 - mostly correct, minor issues
  4 - correct and idiomatic`,
  input: z.object({
    prompt: z.string().describe("The prompt used to generate the completion"),
    completion: z.string().describe("The code completion to evaluate"),
  }),
  output: z.object({
    quality: Scale(1, 4).describe("Quality score from 1 (worst) to 4 (best)"),
  }),
});

await Arbiter.create({
  repo: "your-org/code-completions",
  signature,
  model: "together_ai/openai/gpt-oss-120b",
});
```
</CodeGroup>

Run it to push your Arbiter to Modaic Hub.

<CodeGroup>
```bash python
uv run python create_arbiter.py
```

```bash typescript
npx tsx create_arbiter.ts
```

</CodeGroup>

<Info>
  **Output fields must be discrete** — use `modaic.Scale` (an integer range),
  `modaic.Enum` (a fixed set of choices), or a `Literal`. Arbiters need a finite
  output space to calibrate against. **Always call `.as_arbiter()` before
  `push_to_hub`** — otherwise the repo will not be recognized as an Arbiter on
  Modaic Hub.
</Info>

## 4. Run your Arbiter

Load the deployed Arbiter from Modaic Hub and call it. Inputs must match the input field names on your signature.

<CodeGroup>
```python run_arbiter.py
from modaic import Arbiter

arbiter = Arbiter("your-org/code-completions")

result = arbiter(
    prompt="Print hello world",
    completion="print('Hello World')",
)

print("quality:   ", result.output.quality)
print("reasoning: ", result.reasoning)
print("confidence:", result.confidence)
```

```typescript run_arbiter.ts
import { Arbiter } from "modaic";

const arbiter = new Arbiter("your-org/code-completions");

const result = await arbiter.predict({
  prompt: "Print hello world",
  completion: "print('Hello World')",
});

console.log("quality:  ", result.output?.quality);
console.log("reasoning:", result.reasoning);
```
</CodeGroup>

That's it! Your Arbiter is ready to go.
