`ModaicClient` is the thin Python client for the Modaic REST API. Use it to run arbiters, ingest and annotate examples, and poll jobs without pulling in the full SDK.

## Installation

There are two ways to install the client depending on what else you need.

<Tabs>
<Tab title="Full SDK">
Pick this when you are building, training, or bundling programs with DSPy — it includes `ModaicClient` plus `modaic.Predict`, `PrecompiledProgram`, `AutoProgram`, and the batch stack.

```bash
pip install modaic
```

```python
from modaic import ModaicClient

client = ModaicClient(modaic_token="...")
```
</Tab>
<Tab title="Client Only">
Pick this for production services, serverless functions, or restrictive sandboxes where you only need to *call* arbiters. It ships without DSPy or litellm, so cold starts are fast and the dependency surface is minimal.

```bash
pip install modaic-client
```

```python
from modaic_client import ModaicClient

client = ModaicClient(modaic_token="...")
```
</Tab>
</Tabs>

The public API is identical across both packages — only the import path and dependency footprint differ.

## Authentication

The token can be passed explicitly or picked up from the `MODAIC_TOKEN` environment variable:

```python
import os
os.environ["MODAIC_TOKEN"] = "..."

client = ModaicClient()  # resolves token from env
```

## Calling an Arbiter

The preferred pattern is to construct an `Arbiter` directly — it discovers the shared `ModaicClient` automatically, so there's no need to call `client.get_arbiter(...)`.

### Single Call

Use `Arbiter.predict` to run an arbiter on a single input. Inputs are passed as keyword arguments matching the arbiter's signature.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

prediction = arbiter.predict(ticket="My payment failed twice in a row.")

print(prediction.output)
print(prediction.reasoning)
```

Example output:

```text
{'action': 'refund'}
The user reports a duplicate failed charge, which the rubric classifies as a billing issue eligible for refund.
```

### Multiple Examples

Use `Arbiter.predict_all` to run one arbiter against many examples in a single batch job. The call dispatches a server-side batch and, by default, blocks until results are ready.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

results = arbiter.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice in a row."}},
        {"input": {"ticket": "How do I change my plan?"}},
        {"input": {"ticket": "The app crashes when I open settings."}},
    ],
)

for row in results:
    pred = row.predictions[0]
    print(row.example_id, pred.output)
```

Example output:

```text
ex_01HZ9K2F8V {'action': 'refund'}
ex_01HZ9K2F8W {'action': 'faq'}
ex_01HZ9K2F8X {'action': 'escalate'}
```

Pass `wait_for_results=False` to get back a `BatchJob` handle you can poll with `job.status()` or `job.wait(...)`.

### Multiple Arbiters

Use `ModaicClient.predict_all` when you need to **run multiple arbiters against the same set of examples**. All arbiters are dispatched in a single batch job and evaluated concurrently on the server.

```python
from modaic_client import ModaicClient, Arbiter

client = ModaicClient()

triage = Arbiter("your-org/support-triage")
sentiment = Arbiter("your-org/sentiment")

results = client.predict_all(
    examples=[
        {"input": {"ticket": "My payment failed twice in a row."}},
        {"input": {"ticket": "Thanks, your team is amazing!"}},
    ],
    arbiters=[triage, sentiment],
)

for row in results:
    for pred in row.predictions:
        print(row.example_id, pred.arbiter_repo, pred.output)
```

Example output:

```text
ex_01HZ9K2F8V your-org/support-triage {'action': 'refund'}
ex_01HZ9K2F8V your-org/sentiment {'label': 'negative'}
ex_01HZ9K2F8W your-org/support-triage {'action': 'faq'}
ex_01HZ9K2F8W your-org/sentiment {'label': 'positive'}
```

### Online Confidence Scoring

Pass `compute_confidence=True` to `Arbiter.predict` to kick off confidence scoring as soon as the prediction completes. Reading `prediction.confidence` then blocks only on whatever work is left, instead of starting from scratch.

```python
from modaic_client import Arbiter

arbiter = Arbiter("your-org/support-triage")

prediction = arbiter.predict(
    ticket="My payment failed twice in a row.",
    compute_confidence=True,
)

print(prediction.output)
print(prediction.confidence)
```

Example output:

```text
{'action': 'refund'}
0.87
```

<Warning>
Online confidence scoring is billed per call. Only enable `compute_confidence=True` when you need a confidence score in the response path — for offline scoring of stored predictions, use the batch confidence APIs instead.
</Warning>

## Ingest examples

```python
response = arbiter.ingest_examples([
    {"input": {"ticket": "My payment failed twice in a row."}, "ground_truth": "refund"},
    {"input": {"ticket": "How do I change my plan?"}, "ground_truth": "faq"},
])

print(response)
```

Example output:

```text
queued=True example_ids=['ex_01HZ9K2F8V', 'ex_01HZ9K2F8W']
```

## List examples

```python
page = arbiter.list_examples(page=1, page_size=50)

print(page.total, page.page, page.total_pages)
for example in page.items:
    print(example.id, example.ground_truth, example.confidence)
```

Example output:

```text
128 1 3
ex_01HZ9K2F8V refund 0.87
ex_01HZ9K2F8W faq 0.92
...
```

## Annotate examples

```python
response = arbiter.annotate_example(
    "ex_01HZ9K2F8V",
    ground_truth="escalate",
    ground_reasoning="Repeated failures suggest a payment-provider outage, not a refund case.",
)

print(response)
```

Example output:

```text
status='ok'
```
